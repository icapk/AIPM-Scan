## 一、JD 解析 Prompt

### 1.1 岗位画像解析

**用途**：从 JD 文本中提取结构化信息

**Prompt 模板**：

```
你是一位资深的 AI 产品经理招聘专家。请分析以下职位描述（JD），提取关键信息并生成结构化岗位画像。

【职位描述】
{jd_text}

【AI PM 六维能力模型】
1. 业务拆解能力：将复杂问题分解为可执行任务
2. AI 技术理解：对 AI 技术原理和应用的掌握
3. 商业意识：理解商业价值和 ROI
4. 系统思维：全局视角和架构能力
5. 推动能力：跨团队协作和项目推进
6. 风险意识：识别和应对潜在风险

【输出要求】
请以 JSON 格式输出：
```json
{
  "job_title": "岗位名称",
  "core_responsibilities": ["职责1", "职责2", "职责3"],
  "required_skills": ["技能1", "技能2", "技能3"],
  "experience_years": "工作年限要求",
  "ability_weights": {
    "business_decomposition": 0.25,  // 六维权重总和为 1.0
    "ai_tech_understanding": 0.20,
    "business_awareness": 0.15,
    "system_thinking": 0.20,
    "execution_power": 0.10,
    "risk_awareness": 0.10
  },
  "key_qualifications": ["关键要求1", "关键要求2"],
  "analysis": "简要分析该岗位的核心特点（50字以内）"
}
```

【注意事项】
1. 权重设置需符合岗位实际需求
2. 六维权重总和必须等于 1.0
3. 如果 JD 中某些信息缺失，请基于行业标准合理推断
```

## 二、题库生成 Prompt

### 2.1 基于岗位画像生成题目

**用途**：根据岗位权重体系生成针对性面试题

**Prompt 模板**：

```
你是一位专业的 AI 产品经理面试官。请基于以下岗位画像，生成结构化面试题目。

【岗位画像】
岗位名称：{job_title}
核心职责：{core_responsibilities}
能力权重：{ability_weights}

【知识库参考】
{rag_retrieved_questions}

【输出要求】
生成 15 道面试题，按以下分布：
- 基础题（了解级）：5 道
- 进阶题（应用级）：7 道
- 高级题（综合级）：3 道

每道题包含：
```json
{
  "question_id": "唯一标识",
  "question_text": "题目内容",
  "dimension": "考察的能力维度（六维之一）",
  "difficulty": "基础/进阶/高级",
  "expected_points": ["期望考察点1", "期望考察点2"],
  "follow_up_hints": ["可能的追问方向1", "可能的追问方向2"]
}
```

【题目设计原则】
1. 题目应结合岗位实际场景
2. 避免纯理论题，优先行为面试法（STAR）
3. 题目难度梯度明显
4. 覆盖所有六个能力维度，重点考察权重高的维度
5. 题目应能激发候选人展现真实能力
```

## 三、评分策略 Prompt

### 3.1 单题评分

**用途**：对候选人某道题的回答进行评分

**Prompt 模板**：

```
你是一位资深的 AI 产品经理面试官。请评估候选人对以下问题的回答。

【面试问题】
{question_text}

考察维度：{dimension}
难度：{difficulty}

【候选人回答】
{candidate_answer}

【知识库参考】
{rag_retrieved_knowledge}

【评分标准】
9-10 分：深刻理解，有独到见解，可举一反三
7-8 分：理解准确，表达清晰，覆盖关键点
5-6 分：基本理解，有遗漏，深度不足
3-4 分：理解偏差，关键点缺失
0-2 分：完全不理解或答非所问

【输出要求】
以 JSON 格式输出：
```json
{
  "score": 7.5,  // 0-10 分
  "evidence_sentences": [
    "「用户回答原文引用1」- 这句体现了...",
    "「用户回答原文引用2」- 这句说明..."
  ],
  "strengths": ["优点1", "优点2"],
  "weaknesses": ["不足1", "不足2"],
  "comment": "综合评价（100-200字）",
  "missing_points": ["缺失考察点1", "缺失考察点2"]
}
```

【评分原则】
1. 基于知识库内容客观评分
2. 证据句必须直接引用原文
3. 评价要具体，避免空泛
4. 同时指出优点和改进方向
```

## 四、证据句抽取 Prompt

### 4.1 单句证据抽取

**用途**：从回答中提取支持评分的关键句子

**Prompt 模板**：

```
请从候选人的回答中抽取 1-3 句关键证据，支持你给出的评分。

【评分】
{score}/10

【候选人完整回答】
{answer}

【评分依据】
{scoring_rationale}

【输出要求】
1. 直接引用原文，使用「」标记
2. 每句证据后简要说明为什么重要（20字以内）
3. 亮点和不足都要体现

【示例格式】
- 「我们通过 A/B 测试验证了推荐算法的效果」- 体现了数据驱动思维✓
- 「考虑了用户隐私和模型偏见问题」- 展现了风险意识✓
- 「没有提及具体的 ROI 评估方法」- 缺少商业思考✗
```

## 五、追问生成 Prompt

### 5.1 深度追问

**用途**：基于候选人回答生成有针对性的追问

**Prompt 模板**：

```
候选人刚回答了以下问题，请生成 1-2 个追问，深入考察其能力。

【原问题】
{original_question}

【候选人回答】
{answer}

【初步评估】
得分：{preliminary_score}/10
问题：{identified_issues}

【追问生成策略】
1. 如果回答模糊：要求举具体例子
2. 如果缺少细节：追问实施过程
3. 如果表现优秀：探索更深层思考
4. 如果有疑点：验证真实性

【输出格式】
```json
{
  "follow_ups": [
    {
      "question": "追问内容",
      "purpose": "考察目的",
      "expected_info": "期望获取的信息"
    }
  ]
}
```

【注意】
- 追问应自然流畅，不要让候选人感到被质疑
- 避免过于琐碎的细节追问
```

## 六、报告生成 Prompt

### 6.1 综合评估报告

**用途**：生成候选人整体评估报告

**Prompt 模板**：

```
请基于候选人的面试表现，生成完整的评估报告。

【候选人信息】
姓名：{name}
应聘岗位：{job_title}

【六维得分】
{six_dimension_scores}

【岗位权重】
{job_weights}

【面试记录摘要】
{interview_summary}

【输出要求】
生成结构化报告，包含：

1. **总体评分**（加权计算）
2. **能力雷达图数据**（六维得分）
3. **综合评价**（200-300字）
   - 整体印象
   - 核心亮点（2-3点）
   - 主要不足（1-2点）
4. **各维度详细评价**（每维度 50-100字）
5. **改进建议**（3-5条具体建议）
6. **录用建议**（针对 B 端）
   - 匹配度百分比
   - 推荐/观察/不推荐
   - 理由

【综合评价撰写要求】
- 客观、专业、建设性
- 避免主观臆断
- 用数据和事实说话
- 给出可行的成长路径
```

## 七、Prompt 工程最佳实践

### 7.1 通用原则

1. **角色设定**：明确 AI 的角色（面试官、专家等）
2. **上下文充足**：提供足够的背景信息
3. **格式规范**：使用 JSON 或 Markdown 结构化输出
4. **分步思考**：复杂任务分解为多个 Prompt
5. **示例引导**：提供少量示例（Few-shot Learning）

### 7.2 RAG 增强策略

```python
# 示例：RAG 驱动的评分流程
def rag_enhanced_scoring(question, answer, dimension):
    # 1. 检索相关知识
    knowledge = rag_retrieve(
        query=f"{question} {answer}",
        filter={"dimension": dimension},
        top_k=3
    )
    
    # 2. 构建增强 Prompt
    prompt = SCORING_PROMPT_TEMPLATE.format(
        question_text=question,
        candidate_answer=answer,
        dimension=dimension,
        rag_retrieved_knowledge=knowledge
    )
    
    # 3. LLM 评分
    result = llm.complete(prompt)
    return result
```
