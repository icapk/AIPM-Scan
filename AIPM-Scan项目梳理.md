## 三、技术方案

### 3.1 技术栈选型

| 组件 | 选型 | 理由 |
|-----|------|------|
| 后端框架 | FastAPI (Python) | 现代框架，自动生成 API 文档 |
| 前端 | HTML + Tailwind + JS | 无框架依赖，快速开发 |
| LLM | DeepSeek API | 国产、便宜（约 ¥50/月） |
| 知识库 | RAGFlow | 已熟悉，免费自托管 |
| 数据库 | SQLite | 轻量，无需额外配置 |

### 3.2 核心 API

1. **JD 解析** `/api/parse-jd` - 输入 JD → 输出岗位画像 + 六维权重
2. **题库生成** `/api/generate-questions` - 基于权重检索匹配题目
3. **能力评估** `/api/evaluate-answer` - 六维评分 + 证据句抽取

### 3.3 成本预估

| 项目 | 月成本 |
|-----|-------|
| DeepSeek API | ¥50-100 |
| RAGFlow | ¥0（本地部署） |
| 服务器 | ¥0（本地开发） |
| **合计** | **≤ ¥150/月** |

## 五、AI（Gemini）的核心判断与建议

### 5.1 项目可行性判断

| 维度 | 判断 | 理由 |
|-----|------|------|
| **技术可行性** | ✅ 高 | RAG + LLM 技术成熟，RAGFlow 已熟悉 |
| **时间可行性** | ✅ 可控 | 15天核心开发 + 缓冲，每天6小时够用 |
| **预算可行性** | ✅ 充裕 | 实际成本 ≤ ¥150/月，远低于 ¥500 预算 |
| **求职支撑度** | ✅ 高 | 能展示产品思维 + AI 技术应用能力 |

### 5.2 战略建议

1. **聚焦垂直深度，而非广度**
   - 不要试图覆盖所有职位，只做 AI PM 一个垂直赛道
   - 用专业性对抗通用产品的广度

2. **RAG 是核心技术壁垒**
   - RAG 知识库提升评估准确性、稳定性、可解释性
   - 这是与竞品的关键差异化点

3. **MVP 阶段只做核心功能**
   - ✅ 必须：JD 解析、题库生成、能力评估、简单前端
   - ❌ 暂缓：用户系统、B 端功能、视频面试

4. **面试展示三大亮点**
   - RAG 增强评估（技术深度）
   - 六维能力模型（产品设计）
   - 证据句可解释（用户价值）

### 5.3 风险提示

| 风险 | 概率 | 应对策略 |
|-----|------|---------|
| API 调用失败 | 中 | 增加重试 + 降级策略 |
| 评估不准确 | 中 | 优化 Prompt + 丰富知识库 |
| 时间不够 | 低 | 砍掉非核心功能 |

### 5.4 成功标准

- 3 个核心 API 可用
- 能完整演示面试流程
- 能应对技术问题追问
- 代码可理解可讲解
