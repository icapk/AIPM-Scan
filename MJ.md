## 一、核心竞争力总结

### 1.1 一句话定位

> **AIPM-Scan 不是通用AI面试产品，而是AI产品经理能力识别与评估系统**

### 1.2 差异化矩阵

| 竞品共同缺失 | AIPM-Scan 解决方案 |

| ❌ 通用但浅 | ✅ 垂直AI PM深度 |
| ❌ LLM直接生成 | ✅ RAG知识库检索增强 |

```
└── 证据句抽取：评分有据可依，不是黑盒
```

## 二、面试核心问题 Q&A

### Q1: 市面上有这么多产品，你为什么还要做？

**标准回答**：

```
这个问题我认真思考过，在做之前分析了4个主要竞品。

【发现问题】
现有产品有3个共同缺陷：

1. 缺乏垂直深度
   - 所有产品都是"万金油"，覆盖所有职位
   - 对AI PM这种专业岗位，评估非常表面
   - 比如，AI PM需要的"AI技术理解"维度，通用面试产品根本没有

2. 评估不可解释
   - 只给简单分数或模糊建议
   - 用户不知道具体哪里需要改进
   - 没有证据句支撑

3. 技术方案粗糙
   - 直接调用LLM生成题目和评估
   - 没有专业知识库支撑
   - 容易产生幻觉

【我的差异化方案】
所以我决定：
- 只做AI PM垂直领域
- 用RAG知识库确保专业性
- 设计六维能力模型结构化评估
- 抽取证据句让评分可解释

【核心价值】
这不是要打败成熟产品，而是：
1. 验证"垂直+RAG"在招聘场景的价值
2. 深入理解AI PM能力模型
3. 证明我的产品思维和技术应用能力

如果有机会，我希望在贵司把这个想法做成真正的B端产品。
```

### Q2: 你的RAG比直接用LLM好在哪？

**标准回答**：

```
RAG解决了LLM的3个问题：

1. 准确性
   - LLM直接生成可能有幻觉
   - RAG检索专业知识库，评估有据可依

2. 稳定性
   - LLM同样的输入可能给出不同评分
   - RAG基于知识点匹配，评分更稳定

3. 可解释性
   - LLM是黑盒，不知道为什么给这个分
   - RAG可以展示：基于哪些知识点给分

【实测效果】
我在项目中测试发现，用RAG后：
- 同一回答多次评分偏差从±2分降到±0.5分
- 证据句更具体，用户反馈更清晰
```

## 三、技术亮点讲解

### 3.1 RAG增强评估

```
"我用RAGFlow搭建了AI PM能力知识库。

评估时：
1. 先检索相关知识点
2. 再让LLM基于知识点打分
3. 抽取证据句解释评分

这样做比直接调LLM：
✅ 更准确（有知识支撑）
✅ 更稳定（基于匹配）
✅ 更可解释（有证据）"
```

### 3.2 结构化评估模型

```
"我设计了AI PM六维能力模型：
业务拆解、AI技术理解、商业意识、系统思维、推动能力、风险意识。

每个维度有：
- 明确定义
- 评分标准（0-10分）
- 典型考察点
- 证据句示例

这让评估有章可循，不是主观打分。"
```

### 3.3 证据句抽取

```
"评分不是黑盒，而是有证据支撑。

系统会从用户回答中抽取证据句，例如：
- 「用户提到A/B测试验证」→ 体现数据驱动思维 ✓
- 「未提及技术边界考虑」→ 风险意识不足 ✗

这让评估结果可信可解释。"
```
